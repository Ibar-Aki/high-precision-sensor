1. 実証の目的

高精度傾斜計と机水平ガイドについて、以下を実運用に近い条件で検証する。

1

精度：参照器に対して許容差内か

2

再現性：同条件で結果のばらつきが小さいか

3

実運用性：作業時間・再作業率・失敗率が許容範囲か

2. 検証仮説

仮説①：計測精度

高精度傾斜計は静止状態で参照器との差が ±0.3° 以内に収まる

仮説②：収束性能

机水平ガイドは 2回以内の再計測で水平閾値内へ到達できる

仮説③：オフライン動作

オフライン状態でも主要導線（起動→計測→結果確認）は継続可能

3. 試験設計

3.1 端末と環境

📱 端末

iPhone 12 / 14 / 16 系を各1台以上（iOS最新版）

🔧 設置条件

安定した基準台（石定盤 or 同等）＋微傾斜治具（0.1°単位）

📐 参照器

校正済みデジタル傾斜計（分解能 0.1° 以下）

📶 通信条件

オンライン / オフライン（機内モード相当）を切替

3.2 試験シナリオ

1

標準計測シナリオ

起動 → 権限許可 → 静止計測 → 値記録

2

キャリブレーションシナリオ

1点校正・2点校正をそれぞれ実施し、補正前後の差分を評価

3

机水平調整シナリオ

故意に傾けた机で、ガイドに従い調整。収束回数と時間を計測

4

異常系シナリオ

権限拒否、縦向き違反、オフライン起動、設定保存失敗（容量逼迫模擬）

4. 評価指標（KPI）

指標 計測方法
計測精度 |app値 - 参照器値| の平均 / 95パーセンタイル
再現性 同一条件下の標準偏差
収束性能 目標閾値到達までの再計測回数・総作業時間
運用性 試行失敗率 ＋ オペレータ主観評価（5段階：わかりやすさ・再現しやすさ）
信頼性 オフラインシナリオ成功率 / クラッシュ・フリーズ発生率

Go/No-Go判定、改善項目のリストアップ

9. リスクと対策

R1

端末個体差で結果がぶれる

→ 複数端末で同一シナリオを必ず実施

R2

作業者手順差で結果がぶれる

→ 事前に統一SOPを配布し、読み合わせを実施

R3

参照器の信頼性不足

→ 校正履歴ある参照器を使用、ゼロ点確認を毎日実施

R4

試験中断（通信・電池）

→ オフライン前提手順を準備し、予備端末を確保
