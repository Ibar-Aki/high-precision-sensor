# 本アプリ実証試験（PoC）計画

作成日時: 2026-02-24 19:54:14 +09:00  
作成者: Codex＋GPT-5  
更新日: 2026-02-24

## 1. 実証の目的

高精度傾斜計と机水平ガイドについて、以下を実運用に近い条件で検証する。

1. 精度: 参照器に対して許容差内か
2. 再現性: 同条件で結果のばらつきが小さいか
3. 実運用性: 作業時間・再作業率・失敗率が許容範囲か

## 2. 検証仮説

1. 高精度傾斜計は静止状態で参照器との差が ±0.3° 以内に収まる
2. 机水平ガイドは 2 回以内の再計測で水平閾値内へ到達できる
3. オフライン状態でも主要導線（起動→計測→結果確認）は継続可能

## 3. 試験設計

### 3.1 端末と環境

- 端末: iPhone 12 / iPhone 14 / iPhone 16 系を各1台以上（iOS最新版）
- 設置条件:
  - 安定した基準台（石定盤または同等）
  - 微傾斜を再現できる治具（0.1°単位）
- 参照器:
  - 校正済みデジタル傾斜計（分解能 0.1° 以下）
- 通信条件:
  - オンライン / オフライン（機内モード相当）を切替

### 3.2 シナリオ

1. 標準計測シナリオ
- 起動、権限許可、静止計測、値記録

2. キャリブレーションシナリオ
- 1点校正、2点校正をそれぞれ実施し、補正前後差分を評価

3. 机水平調整シナリオ
- 故意に傾けた机で、ガイドに従って調整し収束回数と時間を計測

4. 異常系シナリオ
- 権限拒否、縦向き違反、オフライン起動、設定保存失敗（容量逼迫模擬）

### 3.3 試行回数

- 各端末 × 各シナリオ × 10試行（最低）
- 合計試行数目安: 3端末 × 4シナリオ × 10 = 120試行

## 4. 評価指標（KPI）

1. 計測精度
- `|app値 - 参照器値|` の平均 / 95パーセンタイル

2. 再現性
- 同一条件下の標準偏差

3. 収束性能（机水平）
- 目標閾値到達までの再計測回数
- 到達までの総作業時間

4. 運用性
- 試行失敗率（計測不能・誤操作・中断）
- オペレータ主観評価（5段階）: わかりやすさ、再現しやすさ

5. 信頼性
- オフラインシナリオ成功率
- 途中クラッシュ/フリーズ発生率

## 5. 合否基準（Go / No-Go）

### Go 条件

1. 高精度傾斜計の 95パーセンタイル誤差が ±0.5° 以内
2. 机水平ガイドの 80%以上が 2回以内再計測で閾値到達
3. 全シナリオの成功率が 95%以上
4. 重大運用障害（手順中断級）が 0件

### No-Go 条件

1. 端末依存で再現する未解決不具合が残る
2. 権限拒否/オフライン時に復旧不能ケースが 5%以上
3. 作業者評価が平均 3.0 未満

## 6. 実施体制

1. 試験責任者: 1名（全体進行、判定）
2. 実験担当: 2名（計測実施、記録）
3. 記録/分析担当: 1名（データ集計、統計処理）
4. 開発担当: 1名（不具合一次解析）

## 7. スケジュール（例: 2週間）

1. Day 1-2: 試験環境準備、端末初期化、治具確認
2. Day 3-6: 標準/校正シナリオ実施
3. Day 7-9: 机水平/異常系シナリオ実施
4. Day 10-11: データ分析、再試験
5. Day 12-14: 判定会、改善項目確定

## 8. 記録フォーマット（最小）

- `session_id`
- `app_type`（tilt / table-level）
- `device_model`, `ios_version`
- `scenario`
- `settings_snapshot`
- `reference_value`, `measured_value`
- `elapsed_time_sec`
- `result`（pass/fail）
- `failure_reason`（該当時）

## 9. リスクと対策

1. 端末個体差で結果がぶれる
- 対策: 複数端末で同一シナリオを必ず実施

2. 作業者手順差で結果がぶれる
- 対策: 事前に統一SOPを配布し、読み合わせを実施

3. 参照器の信頼性不足
- 対策: 校正履歴のある参照器を使用し、ゼロ点確認を毎日実施

4. 試験中断（通信・電池）
- 対策: オフライン前提手順を準備し、予備端末を確保
